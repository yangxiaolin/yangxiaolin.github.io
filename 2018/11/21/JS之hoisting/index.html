为什么

在执行函数时可以拆分为两个步骤（1、进入EC(execution context)并创建VO(variable object)。2、执行函数语句）。
在例1里，在执行函数test后，进入EC, 创建一个VO, VO如下:

可能会感到很疑惑，不是也声明了一个变量v吗？为什么v的值不是10？
在规范里有这么一段话：

If there is already a property of the variable object with
the name of a declared variable, the value of the property and its attributes are not changed.

即如果VO里已经有了一个属性和 变量声明语句创建的变量 相同时，VO里该属性的值不会发生改变。

进入EC并创建完VO后，开始一行行执行代码。这就是为什么例1会打印1的原因。因为在这个时候VO里的v确实是1，不过在执行到
var v = 10这行时，VO里的v就被赋值为10了。

在例2里又稍微有点不同，因为例2里是声明了一个和变量名称相同的函数

在规范里有这么一段话：

If the variable object
already has a property with this name, replace its value and attributes

即如果VO里已经有了一个属性和 声明的函数名称 相同时，VO里该属性的值就会被该函数覆盖掉。

总结

在执行函数时会有如下几个步骤：

  • 将该函数压入执行栈
  • 进入该EC，创建一个VO
  • 将函数的参数放到VO里并设定好值，如果没有值，就设为undefined
  • 如果执行的函数体里有函数声明，则将该函数也放到VO里，有和该函数同名的属性就覆盖
  • 如果执行的函数体里有变量声明，则将变量放到VO里，如果有同名的属性就忽略掉声明的变量

其实就一句话，在VO里，声明的函数会覆盖与其同名的属性，声明的变量如果有与其同名的属性，则会忽略声明的变量
