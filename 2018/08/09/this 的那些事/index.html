在You-Dont-Know-JS得出的几条关于this结论

  • Default Binding(默认绑定？)
    就是说 在一个函数中使用了 this, 但是没有为 this 绑定对象. 这种情况下, 非严格默认, this 就是全局变量 Node 环境中的 global, 浏览器环境中的 window

  • Implicit Binding(隐式绑定)
    使用 obj.foo() 这样的语法来调用函数的时候, 函数 foo 中的 this 绑定到 obj 对象。
    注意隐式绑定时this丢失的情况

  • Explicit Binding(显式绑定)
    就是这些情况：foo.call(obj, …), foo.apply(obj,[…]), foo.bind(obj,…)

  • Hard Binding(硬绑定?)

  • API Call “Contexts”(API调用上下文)

    这里是因为Array.prototype.forEach函数的第二个参数是指定函数运行时的this，
    戳👇这里

  • new Binding(构造绑定)

    new foo()这种情况, 无论 foo 是否做了绑定, 都要创建一个新的对象, 然后 foo 中的 this 引用这个对象

总结

判断this的指向可以使用下面👇几个规则

  • 函数调用前有 new 吗(new foo())？即构造绑定，如果有，this指向新构造的对象

  • 函数调用时有 call, apply, bind 吗(foo.call(obj))？如果有，this指向传递的那个对象(如果传递的对象为null、undefined, this为window/global)

  • 隐式绑定式调用函数吗(obj.foo())？this指向为函数隐式绑定的对象

  • 否则，严格模式下 this 为 undefined, 非严格模式指向 window/global
